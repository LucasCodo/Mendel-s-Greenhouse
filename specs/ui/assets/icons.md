# Icon Catalog

## Purpose

Icons support fast recognition of systems, traits, and actions. They must remain readable at small sizes.

## System Icons

| Icon | Meaning |
| ---- | ------- |
| Gene | Genetic system or gene detail |
| Allele | One inherited variant |
| Phenotype | Visible trait |
| Genotype | Genetic string |
| Contract | Client objective |
| Collection | Discovery catalog |
| Shop | Upgrades and purchases |
| Analyzer | Genetic analyzer |
| Greenhouse | Storage and plant management |
| History | Past crosses |
| Settings | Preferences |

## Action Icons

| Icon | Meaning |
| ---- | ------- |
| Crossbreed | Start cross |
| Store | Keep plant |
| Sell | Remove for credits |
| Deliver | Submit to contract |
| Inspect | Open details |
| Simulate | Preview cross |
| Compare | Compare parents |
| Lock | Unavailable |
| Unlock | Newly available |

## Trait Icons

| Trait | Icon Direction |
| ----- | -------------- |
| Flower color | Simple flower head |
| Leaf width | Leaf silhouette |
| Stem height | Vertical stem marker |
| Petal pattern | Petal with spots |

## Icon Rules

- Use consistent stroke weight or silhouette weight.
- Pair icons with text for unfamiliar systems.
- Do not use icons that imply unsupported mechanics.
- Contract type icons must match contract colors from [../design-system.md](../design-system.md).
